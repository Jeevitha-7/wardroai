# WardroAI Constitution

## Principles

1. WardroAI must run offline on CPU.
2. Image analysis must not depend on cloud APIs.
3. User images and local analysis history must stay local by default.
4. Feature changes should include clear acceptance criteria.
5. Documentation must be updated when user-visible behavior changes.

## Quality Gates

- `npm run lint`
- `npm test`
- Secret scanning before sharing release branches

## Governance

This constitution guides implementation and review decisions for WardroAI features.
